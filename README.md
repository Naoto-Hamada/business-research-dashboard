# Business Research Dashboard

「このサービス、競合と比べて何が強いの？」を、  
**調査・整理・保存・可視化まで一気に自動化**するためのツールです。


非エンジニアでも、
- 1回の指示で
- 比較しやすい形に整理され
- 後から見返せるダッシュボードが残る 

状態を作れます。



## これは何か

普通の調査は、次のようになりがちです。

- 調べるたびに観点がバラバラ
- メモが散らばって後で使えない
- 「この数字どこから？」が追えない
- 毎回ゼロからやり直しになる

このプロジェクトは、上の課題をなくすために作っています。

- 調査観点を固定（顧客・価値・仕組み・不満・財務など）
- 根拠リンク付きで保存
- スプレッドシートに蓄積
- ダッシュボードで比較表示


## 「普通のDeep Researchだけ」で十分では？

Deep Research 自体は強力ですが、実務で使うと次が課題になりやすいです。

1. 出力形式が毎回揺れる
2. チームで再利用しづらい
3. 根拠・数字の追跡が手作業
4. 過去案件との横比較が面倒

このツールの優位性は、**調査後の運用まで設計されていること**です。

- 出力が最初から構造化される
- 根拠と仮定を明示できる
- ダッシュボードで同じ見方で比較できる
- 次の案件で再利用できる


## できること

- 競合調査の自動実行
- BMC（9ブロック）と詳細トピック整理
- 財務シミュレーション（売上・コスト・利益）
- 根拠リンク（`refs`）または仮定（`assumption`）の明示
- 調査完了後、対象ダッシュボードを自動で開く


## 前提条件（必須）

- Claude Code が使えること
- Google アカウント
- ローカルに `clasp`, `jq`, `curl` が入っていること



## 最短セットアップ（1コマンド）

### Step 1. GAS Interpreter2 を設定

まずここから設定してください。  
[GAS Interpreter2 の設定ページ](https://chatgpt-lab.com/n/n290db3448512)

---

### Step 2. セットアップコマンドを実行

以下コマンドを実行すると、次に記載されているものが自動で完了します。

- スプレッドシート作成
- GAS反映
- Webアプリ公開
- Webhook secret 登録
- 環境変数設定

```bash
bash plugins/business-research/scripts/setup_business_research.sh
```

補足:
- `WEBHOOK_SECRET` を指定しない場合、セットアップ時に自動生成されます。
- セットアップ中に Web アプリへ secret を登録し、Claude 環境変数も設定します。

---
### Step 3. 調査コマンドを実行

次のコマンドを実行すると、自動で調査が実行されます。

```text
/business-research
```

サービス名が決まっている場合でも、曖昧な場合でも実行できます。
調査完了後は、対象ダッシュボードURLが必ず出力されます。  
`open/xdg-open` が利用できる環境では自動でブラウザが開きます。

```text
MyFitnessPalを調査して
```

```text
食事記録と運動連携ができるダイエット支援サービスを調べて
```



## 実行フロー（内部でやっていること）

1. 作成
  - 対象と競合を定義し、調査データを作る

2. チェック
  - 必須項目の抜け漏れ
  - 根拠IDの整合
  - 財務の計算整合（売上・コスト・利益）

3. 修正
  - 根拠不足は `refs` を補う
  - 直接根拠がない数値は `assumption` を補う

4. 送信
  - Webhook に保存

5. 表示
  - `?projectId=` 付きダッシュボードを自動で開く



## 品質保証

- 構造品質
  - `projectId` と `businessId` の一意性をチェック
  - `role=調査対象` が1件あることをチェック
  - 各事業で `顧客/価値/仕組み/不満/直近の動き` が揃っていることをチェック
  - `bmcDetails` の9ブロックが埋まっていることをチェック

- 根拠品質
  - 各 `refs` が実在する `sourceId/newsId/quoteId` に紐づくことをチェック
  - ダッシュボード上で根拠リンクを辿れることをチェック
  - 根拠がない記述は `assumption`（仮定）として明示されていることをチェック

- 数値品質（財務）
  - `revenue.items` の合計と `revenue.summary` の整合をチェック
  - `cost.items` の合計と `cost.summary` の整合をチェック
  - `profit = revenue - cost` の整合をチェック
  - 単位（円/月、円/年、%、倍率）の混在・不整合をチェック
  - 異常値（極端な利益率、マイナス売上など）をチェック


## カスタマイズ方法

このプロジェクトは、基本的に Claude Code へ指示してカスタマイズできます。  
下のように「目的」「対象ファイル」「期待する結果」をセットで伝えるとスムーズです。  
下記はカスタマイズしたい際の指示例です。

### 1. 調査項目を変えたい場合
```text
顧客セグメントの調査を強化したいです。
課題の深掘り項目を追加して、送信前チェックにも必須項目として入れてください。
```

### 2. ダッシュボードの見た目を変えたい場合
```text
競合比較テーブルの文字サイズを少し大きくして、
重要な行（売上・利益）だけ背景色を強調してください。
```

### 3. 保存項目（スプレッドシート列）を増やしたい場合
```text
事業シートに「ターゲット地域」列を追加し、
Webhookで受け取った値を保存し、ダッシュボードでも表示してください。
```


## よくある失敗と対処

1. `command not found: clasp`
- `clasp` が未インストールです。インストール後に `clasp login` を実行して再試行してください。

2. `clasp create` や `clasp push` が失敗する
- Google アカウントの認証切れ/権限不足が主因です。`clasp login` を再実行してから再試行してください。

3. セットアップ後にダッシュボードが開けない
- セットアップの出力に出る `Webhook URL` をブラウザで直接開いて確認してください。
- 403/404 の場合、デプロイが途中失敗しているため再実行してください。

## セキュリティ運用メモ

- `WEBHOOK_SECRET` は外部共有しないでください。
- `.env` やローカル設定値は Git にコミットしないでください（`.gitignore` で除外）。
- 画面キャプチャ共有時は URL 内の secret/ID が写っていないか確認してください。
